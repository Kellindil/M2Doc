Services and attributes for the DataBase concept

Une base de donnée dans le studio est un concept groupant

  • des tables
  • des shémas
  • des séquences
  • des définitions de librairie de type utilisateurs.

Une base de donnée possède des attributs et des services qui sont décrits ci dessous.
Dans les exemples employés dans la table ci dessous, la variable db pointe vers une base de donnée.

Attributes

Name Returned type M2Doc Syntaxe Description Example
ID String {m:db.ID} returns the identifier of the database the database identifier: {m:db.id}
comments String {m:db.comments} returns the comments stored on the database comments on the database are :{m:db.comments}
techID String {m:db.techID} returns the technical id of the database
name String {m:db.name} returns the name of the database
tables String {m:db.tables} returns the list of tables directly stored in the database (not in it's schemas)
sequences List of Sequences {m:db.sequences} returns the list of sequences directlry stored in the database (not in it's schemas)
usedLibraries List of user defined libraries {m:db.usedLibraries} returns the list of native type libraries used by the database
url String {m:db.url} returns the URL of the database
schemas List of Schemas {m:db.schemas} returns the list of schemas of the database
defines List of user defined types libraries {m:db.defines} returns the list of user defined type libraries

Services

Name Returned type M2Doc Syntaxe Description Example
all the tables List of tables {m:db.allTables()} returns the set of all tables including the tables in schemas and referenced through foreignkeys the list of database tables : {m:for table | db.allTables()}table {m:table.name},{m:endfor}
all the sequences List of sequences {m:db.allSequences()} returns the set of all sequences related to a table in the result of the allTable() service. the list of database sequences : {m:for sequence | db.allSequences()}sequence {m:sequence.name},{m:endfor}
local tables String {m:db.tables()} returns the list of tables directly stored in the database and its schemas
local sequences List of Sequences {m:db.sequences()} returns the list of sequence directly stored in the database and it's schemas
typeLibraryName String {m:db.typeLibraryName()} Returns the name of the types library used. This name contains the name of the database's engine vendor for physical models.
